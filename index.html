记一次肯德基会话

今天跟一个前同事聊了聊，最近他忙着创业，好久不见，聊了很多。

创业

他目前在创业，所以我们少不了聊这个话题。其中聊了很多关于他目前的这个项目，O2O的创业方向。目前O2O这个方向是很火热，但是热并不代表创业就很容易。以前读书的时候，我可以一腔热血，两眼放光的说我以后一定要去创业，创出属于自己的一片天，但是实际上，出了社会才知道，不是你想的那么简单。

同事是一个比较有想法的人，也敢于付出实践，这点我很钦佩。如果说你下定决心去做一件事，那一定要坚持去把它做好。创业需要的也就是这种决心，不给自己留后路，一路走到底。为创业付出也是巨大的，自己为自己打工，经济压力大，工作强度也大，而且各个方面还需要自己操心，所以一般的人可能都是嘴皮上嚷嚷着要创业，实际上却永远走不出那一步。

其实创业对于我来说，也一直是个梦。找工作的时候，我也想过试试一些创业公司，但是自己深入思考就会发现，自己并不适合创业公司：

  • 对于创业公司而言，生存下去是最重要的。所以他们可能需要的是一个能马上能快速搞定项目的熟手，而不是一个需要成长，在这行经验还不够丰富的“潜力股”
  • 创业公司不稳定，不利于新人的成长，北京的创业公司如此多，每天估计都有倒下的吧，我可不想还没进去热身，公司就黄了
  • 我自己目前也缺乏独挡一面的能力，需要一个技术大牛带着打怪升级

成长

除了创业的话题，还聊了一些关于个人成长的。可能最近我也是经历一些事情，思考了很多东西，所以自己比以前的想法也成熟了些。我们之间，还是有很多观点类似的。所以我一直觉得，我现在更适合一个好的平台、一个好的机会去提升自己的能力，让自己慢慢成长为一个能够独当一面的技术，然后再去考虑其他创业的机会。

以我目前的水平，是不能够支撑起一些不合适的东西。我目前的想法就是，找一个好平台、一个好机会，多多提高自己的技术能力。虽然说我们做技术的思维不能仅仅局限于技术本身，但是以我目前的资历，我个人觉得我还不能够支持我去做技术之外的事情。也许我有可能把它做的还不错，但是我追求的是以技术作为突破口，不仅仅局限于技术层面，去做一些更有意义的事情。

其实我们很多做技术的人都有一个很大的问题，就是仅仅将自己的思维局限于技术这个小小的局部，导致思维很局限。一叶障目，不见泰山。

其实技术只是实现价值的工具，我们的最终的目的是使用技术去更好地服务于人，为人们创造价值，今日有感，希望共勉。最后引用人生赢家强东哥说的一句话：

我觉得踏踏实实、能够创造一定的价值，这个很重要。从小的时候开始，我就一直坚持不会把大把时间放在玩上，从上学，到工作，我都是希望每天忙忙碌碌，有事做，让这一生活得有价值。做公司也是如此，只要我认为我做的事情是有价值的、我喜欢的，确确实实能够为这个行业创造一些价值的话，我就会坚信去做，不管是花10年、20年。

commons-lang这货是让我没法造轮子啊

自从参加工作以后，一直有一个想法就是想写一些自己的基础类库，工具类吧啦吧啦之类的，但是越到现在越发现，这个想法是多余了的。越来越觉得Java的强大之处在于你想要的东西或者想造的轮子，在某一天，突然得知原来早就有了现成的十分健全的库可供使用，不得不感叹Java的大而全，也发现自己的积累不足。

这不，commons-lang就是一个，平时用的比较多的API，mark之。

简介

引用官网Apache Commons Lang

Lang provides a host of helper utilities for the java.lang API, notably String manipulation methods, basic numerical methods, object reflection, concurrency, creation and serialization and System properties. Additionally it contains basic enhancements to java.util.Date and a series of utilities dedicated to help with building methods, such as hashCode, toString and equals.

再见，祝好！

今天下午拿到的离职证明，从14年4月入职到15年8月24日，不到一年四个月。虽然很难说出再见，但是天下没有不散的宴席，这一天迟早会来的，只是没有预料到会这么快。自此，我的第一份工作划了个句号，带着极其复杂的心情。

13年9月，与当年那一届的其他应届生一样，开始了自己漫长纠结的求职之旅。由于自己的没心没肺，也可能是准备不足，过程并不是很顺利，但是人人给了我人生中第一个offer。各种阴差阳错，最终来了人人。其中痛苦纠结的过程，只有自己最清楚。

入职之前，我就听说过公司的很多问题，甚至有人说人都走完了，还来这干嘛，那会其实心里也很失望的。对于一个刚入社会的新人，第一个公司竟然是这么个情况，我的心理实在不是滋味
。我还跑到知乎上去提了个问题，这个问题还得到了当时公司内的人回答，让我当时觉得公司还是有很多靠谱的人在做着靠谱的事。慢慢地也就接受了公司不景气这个事实。

入职之后，我觉得自己好幸运，因为我来了一个特别nice的团队，做着一些比较有价值的事情。首先，我们团队还是有很多事情可以做的，我们从无到有搭建一整个广告后台系统，虽然我只是一个打酱油的应届生，但是通过参与后台系统的开发，学到了很多东西。来公司之前，我一行Java代码都没写过，所有东西都是从0开始学习，那会每天白天在公司干活，晚上还回去啃书，觉得自己成长很快。其次，整个团队氛围是很和谐的，琦鸿、宇阳、胜利、阿哥、龙哥、迪哥、波哥、涵仔，玥姐，还有后来的爱春，都是很赞的人，相处都十分愉快。怀念那会每天中午去木桶饭吃饭的日子，还有美岸（要不哥机智用团购，你们还傻傻地原价掏钱）。那会我们会调侃龙哥和玥姐，会八卦波波哥的女神和岳父，会被二春的二给逗的不行，会嘲笑哥的骑车漏风，扯一些扯不清的基情。去年一年，我们没有一个离职的小伙伴，一起度过了很愉快的一年。

毕竟公司不太景气，离职是迟早的事。后来就陆续有小伙伴走了，所以离职也是情理之中。只是觉得自己很幸运，能够找到目前的工作，感恩。怀念以前在人人的日子，感谢那些对我提供过帮助的人，珍惜当前所拥有的一切。加油！祝各位小伙伴事业蒸蒸日上，爱情幸福美满！祝前东家一切都好！

其实，我是一个码农

早就有自己建一个个人博客的想法了，一直因为各种原因没有去执行，主要是因为「懒」。最近公司事情也不太多，因此突然心血来潮，去网上搜了一堆教程后，参考了这篇博文：

Hexo主题Yilia

在此表示感谢。捣腾了一下午，顺带把Mardown学了下，总算是把博客搭建好了，剩下的就等着填充博客内容了（来日方长^_^）。

我来挑刺

毕竟现在已经有了各种博客网站，为什么还要自己搭建一个呢，主要是基于以下几点：

  • 目前的博客网站的阅读体验非常差：广告满屏飞，博客样式和排版十分难看，编辑器十分难用
  • 很多博客不支持Markdown，也不能代码高亮，这对一个coder来说是致命的
  • 博文不能乱写否则会被「和谐」，呵呵，你懂的

自己DIY

自己建博客好处多多啊，就是刚开始可能麻烦点，只要搭建好了，后面就好说，万事开头难嘛，既然是自己建的博客：

  • 可以自己定义各种主题，DIY可玩性更高
  • 作为一个coder，这年头没有个人博客都不好意思说你是coder了
  • 使用markdown排版，最近刚接触到Markdown，只能用四个字来形容了「相见恨晚」，完全秒杀各种编辑器了，让我有了写博客的欲望
  • 自己的博客想些啥就写啥，反正托管到github上，再也不用担心无缘无故被和谐啦

为啥要写博客

从小我语文就不好，作文更是从小就被称为流水账，每次语文考试作为都是及格分，但是这些都不能影响我写博客的热情，好不容易找到一件能让自己好好做的事情呢。

沉下心做事情

我觉得我自己是一个很容易浮躁的人，也因此吃过许多亏。读书的时候，每当我感觉自己有些飘飘然的时候，我就回去自习室或者图书馆，不求能学到东西，只求让自己的内心沉静下来，正如功夫熊猫中所追求的：

inner peace

对于我自己而言，我内心最安静地时候是在写作的时候，因此写博客有利于消除我内心的浮躁，安安静静做点事情。

职业所致

而对于一个靠技术吃饭的coder来说，技术变化日新月异，时不时就会有各种新技术出来，各种新技术需要学习。通过写技术博客这种方式，能够让自己沉下心来学习新技术，总结并思考，记录笔记，能达到事半功倍的效果，就算过一段时间忘了，回头看看自己的博客，沿着当时思考的路径，也能很快再次掌握。

总要记录点啥吧

作为一个工科男，也允许我偶尔文艺下吧。虽然文笔拙略，但是好歹也是自己成长的一个见证，可以记录自己的心路历程，记录自己的成长故事，没有比这个更好地方式了。

时候不早了，现在都凌晨12点20了，就写这么多吧，晚安。
